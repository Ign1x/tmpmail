# TmpMail API Reference
# Base URL: http://127.0.0.1:18081

Authentication

1. Account token
- POST /token returns a bearer token for mailbox operations.
- Use this token for /me, /messages, /messages/:id, and DELETE /accounts/:id.

2. Console API key
- Domain-management endpoints use the backend API key generated for the currently signed-in console user.
- The key is shown only when it is first created or rotated. The server stores only a hash and cannot reveal the previous plaintext value again.
- Use HTTPS on any public network before transmitting the console password, session token, or console API key.
- Send it as: Authorization: Bearer <console-session-token-or-api-key>

Core Endpoints

GET /healthz
- Health probe.

GET /domains
- Without console credentials: returns active public domains.
- With a signed-in console user or console API key: returns the caller's managed domains.
- With an administrator session or admin-role API key: returns all managed domains.

POST /domains
- Console user required.
- Regular users create domains under their own quota.
- Administrators can create and inspect all managed domains.
- Body:
{
  "domain": "mail.example.com"
}

GET /domains/:id/records
- Console user required.
- Regular users can only read DNS records for their own managed domains.
- Returns the CNAME, MX, and TXT records required for managed-domain verification.

POST /domains/:id/verify
- Console user required.
- Regular users can only verify their own managed domains.
- Triggers an immediate DNS verification pass.

POST /accounts
- Create a mailbox account on any active domain.
- Body:
{
  "address": "user@mail.example.com",
  "password": "your_password",
  "expiresIn": 0
}

POST /token
- Body:
{
  "address": "user@mail.example.com",
  "password": "your_password"
}

GET /me
- Bearer token required.

DELETE /accounts/:id
- Bearer token required.
- Can only delete the currently authenticated account.

GET /messages
- Bearer token required.

GET /messages/:id
- Bearer token required.

PATCH /messages/:id
- Bearer token required.
- Body:
{
  "seen": true
}

DELETE /messages/:id
- Bearer token required.

Typical Console Flow

1. Open the configured admin entry in the frontend. The default entry is /admin.
2. On the first access, create the first administrator username and password.
3. Administrators can create more console users and assign domain quotas.
4. Sign in with username + password, then issue or rotate the current user's backend API key if needed.
5. Create a domain with POST /domains.
6. Read DNS records from GET /domains/:id/records.
7. Verify with POST /domains/:id/verify.
8. Once status becomes active, all users can create accounts on that domain.

Typical User Flow

1. Read active domains from GET /domains.
2. Create an account with POST /accounts.
3. Log in with POST /token.
4. Read mailbox data with /me and /messages.
